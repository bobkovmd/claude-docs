# Использование инструментов — Полная документация

## Источник
https://platform.claude.com/docs/en/agents-and-tools/tool-use/

## Обзор

Использование инструментов позволяет Claude вызывать **функции, определённые вами** (клиентские инструменты) или **функции, предоставляемые Anthropic** (серверные инструменты).

- **Клиентские инструменты**: Ваше приложение выполняет их. Claude возвращает `stop_reason: "tool_use"` + блоки `tool_use`.
- **Серверные инструменты**: Anthropic управляет выполнением; результаты появляются напрямую.

### Категории
1. **Инструменты, определённые пользователем (выполняемые на клиенте)**: Вы пишете схему, выполняете код, возвращаете результаты
2. **Инструменты со схемой Anthropic (выполняемые на клиенте)**: bash, text_editor, computer, memory
3. **Инструменты, выполняемые на сервере**: web_search, web_fetch, code_execution, tool_search

### Агентный цикл
1. Отправьте запрос с массивом `tools`
2. Claude отвечает с `stop_reason: "tool_use"` + блоки `tool_use`
3. Выполните каждый инструмент → форматируйте выходные данные как `tool_result`
4. Отправьте новый запрос с исходными сообщениями + ответом ассистента + `tool_result`
5. Повторяйте, пока `stop_reason == "tool_use"`

### Схема определения инструмента
| Параметр | Описание |
|---|---|
| `name` | Должен соответствовать `^[a-zA-Z0-9_-]{1,64}$` |
| `description` | Подробный текст: что, когда, как |
| `input_schema` | JSON Schema для параметров |
| `input_examples` | Необязательные примеры входных объектов |

### Лучшие практики
- Предоставляйте **крайне подробные описания** (3–4+ предложения)
- Используйте `input_examples` для сложных инструментов
- Объединяйте связанные операции с параметром `action`
- Используйте осмысленное пространство имён
- Проектируйте ответы инструментов так, чтобы возвращать только информацию с высокой ценностью

### Параллельное использование инструментов
- Все результаты параллельных инструментов должны быть в **одном сообщении пользователя**
- Используйте системный промпт для поощрения параллельных вызовов

### Инструментальный раннер (SDK)
Автоматизирует агентный цикл вызова инструментов. Доступен на Python, TypeScript, C#, Go, Java, PHP, Ruby.

### Строгое использование инструментов
Установка `strict: true` гарантирует соответствие JSON Schema через семантически ограниченную выборку.

### Серверные инструменты
- Блоки `server_tool_use` с префиксом `srvtoolu_`
- `pause_turn` для длительных ходов
- Фильтрация доменов с `allowed_domains` / `blocked_domains`

### Веб-поиск
Последняя версия: `web_search_20260318`. $10 за 1 000 поисковых запросов.

### Выполнение кода
Песочница Python/Bash. Бесплатно при использовании с web_search или web_fetch.

### Поиск инструментов
Масштабирование до тысяч инструментов с динамическим обнаружением на основе регулярных выражений.

### Инструмент памяти
Директория `/memories` на стороне клиента с операциями CRUD.

### Использование компьютера
Взаимодействие с рабочим столом через скриншоты, мышь и клавиатуру.
