# Промптинг Claude Opus 4.8

## Обзор

Claude Opus 4.8 превосходно справляется с агентной работой на дальние горизонты, интеллектуальной работой, задачами на зрение и памятью. Он хорошо работает «из коробки» с существующими промптами Claude Opus 4.7, но несколько поведенческих паттернов могут потребовать настройки.

## Длина ответа и многословность

- **Калибрует длину ответа на основе воспринимаемой сложности задачи** — более короткие ответы для простых поисков, более длинные для открытого анализа
- Если ваш продукт требует определённой многословности, настройте промпты явно

## Настройка Effort и глубины мышления

Параметр `effort` обменивает интеллект на скорость/стоимость. **Effort важнее для Opus 4.8, чем для любой предыдущей модели Opus.**

| Уровень Effort | Лучше всего для |
|--------------|----------|
| **`max`** | Задачи, требующие высокого интеллекта; может показывать убывающую отдачу или избыточное мышление |
| **`xhigh`** | **Лучший выбор для большинства сценариев программирования и агентнных задач** |
| **`high`** | Минимум для чувствительных к интеллекту сценариев; балансирует токены и интеллект |
| **`medium`** | Чувствительные к стоимости сценарии; жертвует интеллектом |
| **`low`** | Короткие, ограниченные, чувствительные к задержке задачи, не требующие глубокого рассуждения |

### Ключевые поведения
- **Строго соблюдает уровни effort** — при `low`/`medium` ограничивает работу именно тем, что было запрошено
- Риск **недостаточного мышления** на умеренно сложных задачах при `low` effort
- Если наблюдается поверхностное рассуждение, **повысьте effort до `high`/`xhigh`**, а не пытайтесь обойти это промптингом

## Адаптивное мышление

- **По умолчанию мышление ВЫКЛЮЧЕНО** — необходимо явно установить `thinking: {type: "adaptive"}`
- При `max` или `xhigh` effort установите большой максимальный бюджет выходных токенов (начните с **64k токенов**)

## Запуск использования инструментов

- **Предпочитает рассуждение вызовам инструментов** — даёт лучшие результаты в большинстве случаев
- **Увеличение effort увеличивает использование инструментов** — `high`/`xhigh` показывают значительно больше использования инструментов

## Буквальное следование инструкциям

- **Интерпретирует промпты буквально и явно**, особенно при более низких уровнях effort
- **Не** молчаливо обобщает инструкции или выводит невысказанные запросы
- **Преимущество:** Точность и меньше хаоса; лучше для API-сценариев, структурированного извлечения

## Тон и стиль письма

- Склоняется к **прямому, уверенному стилю** с минимальной валидационно-ориентированной формулировкой
- Если голос вашего продукта отличается, пересмотрите промпты стиля

## Контроль порождения субагентов

- **По умолчанию порождает меньше субагентов** — поведение можно направлять через промптинг

## Дизайн и настройки фронтенда по умолчанию

### Стиль по умолчанию (House Style)
- **Фоны:** Тёплый кремовый/беловатый (~`#F4F1EA`)
- **Шрифт отображения:** С засечками (Georgia, Fraunces, Playfair)
- **Акценты:** Курсивные акценты на словах, терракотовый/янтарный

> ⚠️ Это хорошо подходит для редакционных/гостиничных/портфолио сайтов, но выглядит неуместно для дашбордов, инструментов разработчика, финтеха, здравоохранения или корпоративных приложений.
