# Лучшие практики промптинга

## Обзор

Исчерпывающее руководство по инжинирингу промптов для последних моделей Claude: **Claude Fable 5, Claude Mythos 5, Claude Opus 4.8, Claude Opus 4.7, Claude Opus 4.6, Claude Sonnet 4.6 и Claude Haiku 4.5**.

Организовано в три части:
1. **Руководство для конкретных моделей** (Fable 5, Opus 4.8)
2. **Техники для всех текущих моделей** (общие принципы, вывод/форматирование, использование инструментов, мышление, агентные системы)
3. **Замечания по миграции** с более ранних поколений моделей

---

## Общие принципы (Все модели)

### Будьте ясны и прямы

Считайте Claude блестящим, но новым сотрудником, который не знаком с вашими нормами и рабочими процессами.

**Золотое правило:** Покажите свой промпт коллеге с минимальным контекстом. Если он запутается, Claude тоже запутается.

- Укажите желаемый формат вывода и ограничения
- Используйте нумерованные списки/маркированные списки, когда порядок важен
- Явно запрашивайте поведение «выше и за пределами» — не полагайтесь на умозаключения

### Добавьте контекст/мотивацию

Объяснение *почему* помогает Claude обобщать.

### Эффективно используйте примеры (Few-Shot)

- **Релевантные**: Отражайте реальный сценарий использования
- **Разнообразные**: Охватывайте граничные случаи
- **Структурированные**: Оборачивайте в теги `<example>` / `<examples>`
- **Включайте 3–5 примеров** для лучших результатов

### Структурируйте промпты с помощью XML-тегов

Используйте последовательные, описательные имена тегов. Вкладывайте для иерархии.

### Дайте Claude роль

Одно предложение в системном промпте фокусирует поведение и тон.

### Промптинг с длинным контекстом (20k+ токенов)

- **Размещайте длинные данные сверху** — выше запроса, инструкций и примеров
- **Структурируйте документы с помощью XML-тегов**
- **Обосновывайте ответы в кавычках**

## Вывод и форматирование

### Изменения стиля общения

Последние модели стали более **прямыми, обоснованными, разговорными и менее многословными**.

### Контроль формата ответа

1. **Говорите Claude, что ДЕЛАТЬ, а не что НЕ делать**
2. **Используйте индикаторы формата XML**
3. **Соответствуйте стилю промпта желаемому выводу**
4. **Подробные инструкции по форматированию**

### Уход от предзаполненных ответов

Предзаполненные сообщения ассистента на последнем ходе ассистента **больше не поддерживаются** начиная с Claude 4.6 (возвращает ошибку 400).

## Использование инструментов

### Явное указание действий
- ❌ «Можете предложить несколько изменений?» (Claude только предложит)
- ✅ «Измените эту функцию для улучшения её производительности.» (Claude выполнит действие)

### Выбор инструмента

Claude обычно предпочитает рассуждать перед использованием инструментов.

## Мышление

### Расширенное мышление

Включите с помощью `thinking: {type: "enabled", budget_tokens: N}`

### Адаптивное мышление (Claude Opus 4.8+)

- По умолчанию мышление ВЫКЛЮЧЕНО — установите `thinking: {type: "adaptive"}`
- Контролируйте глубину через параметр `effort`
- При `max`/`xhigh` effort установите большой максимальный бюджет выходных токенов (начните с **64k токенов**)
