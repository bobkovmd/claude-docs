# Инжиниринг промптов (Prompt Engineering)

## Обзор
Все техники промптинга — ясность, примеры, XML-структурирование, ролевой промптинг, мышление, цепочки промптов.

## Лучшие практики

### Будьте ясны и прямы
Считайте Claude «блестящим, но новым сотрудником», не знакомым с вашими нормами.

### Добавьте контекст/мотивацию
Объяснение *почему* помогает Claude обобщать.

### Эффективно используйте примеры (Few-Shot)
- Включайте 3–5 примеров
- Релевантные, разнообразные, структурированные в тегах `<example>`

### Структурируйте промпты с помощью XML-тегов
Используйте последовательные, описательные имена тегов с вложенностью.

### Дайте Claude роль
Одно предложение в системном промпте фокусирует поведение и тон.

### Промптинг с длинным контекстом (20k+ токенов)
- Размещайте длинные данные СВЕРХУ
- Структурируйте документы с помощью XML-тегов
- Обосновывайте ответы в кавычках

## Руководства для конкретных моделей

### Claude Fable 5
- Более длинные ходы по умолчанию
- Сильное следование инструкциям
- Обоснование утверждений о прогрессе во время длинных запусков
- Параллельные субагенты

### Claude Opus 4.8
- Калибрует длину ответа в зависимости от сложности задачи
- Effort важнее, чем в любой предыдущей модели Opus
- Буквальное следование инструкциям
- Прямой, уверенный тон

## Инструменты промптинга в Console
1. Генератор промптов — решает «проблему пустой страницы»
2. Шаблоны и переменные — синтаксис `{{variable_name}}`
3. Улучшатель промптов — автоматическое улучшение в 4 шага
