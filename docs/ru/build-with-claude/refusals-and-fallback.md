# Отказы и резервирование (Refusals and Fallback)

## Обзор
Claude Fable 5 включает классификаторы безопасности, которые отклоняют определённые запросы. Отказы возвращаются как HTTP 200 с `stop_reason: "refusal"`.

## Ответ на отказ
```json
{
  "stop_reason": "refusal",
  "stop_details": {
    "type": "refusal",
    "category": "cyber",
    "explanation": "Этот запрос был отклонён, поскольку может способствовать кибервреду."
  }
}
```

### Категории отказов
| `category` | Значение |
|---|---|
| `"cyber"` | Может способствовать кибервреду |
| `"bio"` | Может способствовать биологическому вреду |
| `"frontier_llm"` | Может способствовать разработке конкурирующих AI-моделей |
| `"reasoning_extraction"` | Запрашивает воспроизведение внутреннего рассуждения модели |

## Подходы к резервированию
1. **Серверное резервирование** — Один запрос, API обрабатывает повторную попытку
2. **ПО промежуточного слоя SDK** — Настройте один раз на клиенте
3. **Ручная повторная попытка с резервным кредитом** — Полный контроль

## Серверное резервирование
- Заголовок beta: `anthropic-beta: server-side-fallback-2026-06-01`
- До 3 резервных моделей
- Только отказы классификаторов безопасности запускают резервирование

## Резервный кредит
Избежание двойной оплаты затрат на кэш промптов при повторной попытке отклонённого запроса на другой модели.
