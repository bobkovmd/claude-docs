# Быстрый режим (исследовательская предварительная версия)
Получайте до 2,5 раз больше выходных токенов в секунду от моделей Claude Opus.Быстрый режим обеспечивает до 2,5 раз больше выходных токенов в секунду от Claude Opus 4.8, Claude Opus 4.7 и Claude Opus 4.6 по премиальной цене. Установите `speed: "fast"` вместе с бета-заголовком `fast-mode-2026-02-01` в вашем запросе, чтобы включить этот режим.

Быстрый режим находится в стадии исследовательской предварительной версии. Свяжитесь с вашим менеджером по работе с клиентами, чтобы запросить доступ. Если у вас нет менеджера по работе с клиентами, [присоединитесь к списку ожидания](https://claude.com/fast-mode) для быстрого режима.

Эта функция соответствует требованиям [Zero Data Retention (ZDR)](/docs/ru/build-with-claude/api-and-data-retention) (нулевого хранения данных). Если у вашей организации действует соглашение ZDR, данные, отправленные через эту функцию, не сохраняются после возврата ответа API.

## Поддерживаемые модели

Быстрый режим поддерживается для следующих моделей:

- Claude Opus 4.8 (claude-opus-4-8)
- Claude Opus 4.7 (claude-opus-4-7)
- Claude Opus 4.6 (claude-opus-4-6)

Быстрый режим для Claude Opus 4.8 запускается как исследовательская предварительная версия только в Claude API, включая [Claude Managed Agents](/docs/ru/managed-agents/overview). Он недоступен на сторонних платформах, включая Vertex AI, Amazon Bedrock и Microsoft Foundry.

Быстрый режим для Claude Opus 4.6 объявлен устаревшим с момента запуска Claude Opus 4.8 и будет удалён примерно через 30 дней. После удаления запросы к `claude-opus-4-6` с параметром `speed: "fast"` будут переключаться на стандартную скорость по стандартной цене, а не возвращать ошибку. Перейдите на быстрый режим для Claude Opus 4.8 или Claude Opus 4.7, чтобы сохранить ускорение.

## Как работает быстрый режим

Быстрый режим запускает ту же модель с более быстрой конфигурацией инференса. Интеллект и возможности модели не меняются.

- До 2,5 раз больше выходных токенов в секунду по сравнению со стандартной скоростью
- Преимущества в скорости сосредоточены на показателе «output tokens per second» (выходные токены в секунду), или OTPS, а не на «time to first token» (время до первого токена), или TTFT
- Те же веса модели и поведение (это не другая модель)
- Совместим с [потоковой передачей](/docs/ru/build-with-claude/streaming), где прирост OTPS наиболее заметен

## Базовое использование

```
`client = anthropic.Anthropic()

response = client.beta.messages.create(
    model="claude-opus-4-8",
    max_tokens=4096,

    messages=[
        {"role": "user", "content": "Refactor this module to use dependency injection"}
    ],

print(response.content[0].text)`
```

## Цены

Быстрый режим тарифицируется с применением множителя к стандартным тарифам для каждой модели по всему контекстному окну, включая запросы с более чем 200 тыс. входных токенов. В следующей таблице приведены цены быстрого режима для каждой поддерживаемой модели:

МодельВходВыходClaude Opus 4.8$10 / млн токенов$50 / млн токеновClaude Opus 4.7 / Claude Opus 4.6$30 / млн токенов$150 / млн токенов
Цены быстрого режима суммируются с другими модификаторами цен:

- [Множители кэширования подсказок](/docs/ru/about-claude/pricing#prompt-caching) применяются поверх цен быстрого режима
- Множители [резидентности данных](/docs/ru/manage-claude/data-residency) применяются поверх цен быстрого режима

Полную информацию о ценах см. на [странице цен](/docs/ru/about-claude/pricing#fast-mode-pricing).

## Ограничения скорости

Быстрый режим имеет отдельное ограничение скорости, не связанное со стандартными ограничениями скорости для Opus. Когда ваше ограничение скорости для быстрого режима превышено, API возвращает ошибку `429` с заголовком `retry-after`, указывающим, когда ёмкость станет доступной.

Ответ включает заголовки, которые указывают статус вашего ограничения скорости для быстрого режима:

ЗаголовокОписание`anthropic-fast-input-tokens-limit`Максимальное количество входных токенов быстрого режима в минуту`anthropic-fast-input-tokens-remaining`Оставшиеся входные токены быстрого режима`anthropic-fast-input-tokens-reset`Время сброса лимита входных токенов быстрого режима`anthropic-fast-output-tokens-limit`Максимальное количество выходных токенов быстрого режима в минуту`anthropic-fast-output-tokens-remaining`Оставшиеся выходные токены быстрого режима`anthropic-fast-output-tokens-reset`Время сброса лимита выходных токенов быстрого режима
Ограничения скорости для конкретных уровней см. на [странице ограничений скорости](/docs/ru/api/rate-limits).

## Проверка использованной скорости

Объект `usage` в ответе включает поле `speed`, которое указывает, какая скорость была использована: `"fast"` или `"standard"`. Быстрый режим не переключается незаметно на стандартную скорость при достижении ограничений скорости или нехватке ёмкости (вместо этого вы получите ошибку `429` или `529`), поэтому, когда вы запрашиваете `speed: "fast"` для поддерживаемой модели, значение `usage.speed` равно `"fast"`.

```
`client = anthropic.Anthropic()

response = client.beta.messages.create(
    model="claude-opus-4-8",
    max_tokens=1024,
    speed="fast",
    betas=["fast-mode-2026-02-01"],
    messages=[{"role": "user", "content": "Hello"}],
)

print(response.usage.speed)  # "fast" or "standard"`
```

## Повторные попытки и резервный вариант

### Автоматические повторные попытки

Когда ограничения скорости быстрого режима превышены, API возвращает ошибку `429` с заголовком `retry-after`. SDK Anthropic автоматически повторяют эти запросы до 2 раз по умолчанию (настраивается с помощью `max_retries`), ожидая указанную сервером задержку перед каждой повторной попыткой. Поскольку быстрый режим использует непрерывное пополнение токенов, задержка `retry-after` обычно короткая, и запросы выполняются успешно, как только ёмкость становится доступной.

### Переключение на стандартную скорость

Если вы предпочитаете переключиться на стандартную скорость, а не ждать доступности ёмкости быстрого режима, перехватите ошибку ограничения скорости и повторите запрос без `speed: "fast"`. Установите `max_retries` в `0` для первоначального быстрого запроса, чтобы пропустить автоматические повторные попытки и немедленно завершиться с ошибкой при достижении ограничений скорости.

Переключение с быстрой на стандартную скорость приведёт к промаху [кэша подсказок](/docs/ru/build-with-claude/prompt-caching). Запросы с разными скоростями не используют общие кэшированные префиксы.

Поскольку установка `max_retries` в `0` также отключает повторные попытки для других временных ошибок (перегрузка, внутренние ошибки сервера), следующие примеры повторно отправляют исходный запрос с повторными попытками по умолчанию для таких случаев.

```
`client = anthropic.Anthropic()

def create_message_with_fast_fallback(max_retries=0, max_attempts=3, **params):
    try:
        return client.with_options(max_retries=max_retries).beta.messages.create(
            **params
        )
    except anthropic.RateLimitError:
        if params.get("speed") == "fast":
            del params["speed"]
            return create_message_with_fast_fallback(max_retries=max_retries, **params)
        raise
    except (
        anthropic.APIStatusError,
        anthropic.APIConnectionError,
    ) as error:
        if isinstance(error, anthropic.APIStatusError) and error.status_code < 500:
            raise
        if max_attempts > 1:
            return create_message_with_fast_fallback(
                max_retries=max_retries, max_attempts=max_attempts - 1, **params
            )
        raise

message = create_message_with_fast_fallback(
    model="claude-opus-4-8",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello"}],
    betas=["fast-mode-2026-02-01"],
    speed="fast",
    max_retries=0,
)`
```

## Особенности

- **Кэширование подсказок:** Переключение между быстрой и стандартной скоростью делает кэш подсказок недействительным. Запросы с разными скоростями не используют общие кэшированные префиксы.
- **Поддерживаемые модели:** Быстрый режим поддерживается для Claude Opus 4.8, Claude Opus 4.7 и Claude Opus 4.6. Отправка `speed: "fast"` с неподдерживаемой моделью возвращает ошибку.
- **TTFT:** Преимущества быстрого режима сосредоточены на выходных токенах в секунду (OTPS), а не на времени до первого токена (TTFT).
- **Batch API:** Быстрый режим недоступен с [Batch API](/docs/ru/build-with-claude/batch-processing).
- **Priority Tier:** Быстрый режим недоступен с [Priority Tier](/docs/ru/api/service-tiers).
- **Claude Platform на AWS:** Быстрый режим в настоящее время недоступен на [Claude Platform на AWS](/docs/ru/build-with-claude/claude-platform-on-aws).
