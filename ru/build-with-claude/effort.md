# Effort
Управляйте количеством токенов, которые Claude использует при ответе, с помощью параметра effort, балансируя между полнотой ответа и эффективностью использования токенов.Эта функция соответствует требованиям [Zero Data Retention (ZDR)](/docs/ru/build-with-claude/api-and-data-retention) (нулевого хранения данных). Если у вашей организации действует соглашение ZDR, данные, отправленные через эту функцию, не сохраняются после возврата ответа API.

Параметр effort позволяет управлять тем, насколько охотно Claude расходует токены при ответе на запросы. Вы можете балансировать между полнотой ответа и эффективностью использования токенов в рамках одной модели. Параметр effort доступен для всех поддерживаемых моделей без необходимости указывать бета-заголовок.

Параметр effort поддерживается моделями Claude Fable 5, [Claude Mythos 5](https://anthropic.com/glasswing), Claude Opus 4.8, [Claude Mythos Preview](https://anthropic.com/glasswing), Claude Opus 4.7, Claude Opus 4.6, Claude Sonnet 4.6 и Claude Opus 4.5.

Для Claude Opus 4.6 и Sonnet 4.6 параметр effort заменяет `budget_tokens` в качестве рекомендуемого способа управления глубиной мышления. Сочетайте effort с [адаптивным мышлением](/docs/ru/build-with-claude/adaptive-thinking) (`thinking: {type: "adaptive"}`) для наилучшего результата. Хотя `budget_tokens` всё ещё принимается в Opus 4.6 и Sonnet 4.6, он считается устаревшим и будет удалён в одном из будущих выпусков модели. При уровнях effort `high` (по умолчанию) и `max` Claude почти всегда будет использовать мышление. При более низких уровнях effort он может пропускать мышление для более простых задач.

## Как работает effort

По умолчанию Claude использует высокий уровень effort, расходуя столько токенов, сколько необходимо для получения отличных результатов. Вы можете повысить уровень effort до `max` для достижения абсолютно максимальных возможностей или понизить его, чтобы более экономно расходовать токены, оптимизируя скорость и стоимость при некотором снижении возможностей.

Установка `effort` в значение `"high"` даёт точно такое же поведение, как и полное отсутствие параметра `effort`.

Параметр effort влияет на **все токены** в ответе, включая:

- Текстовые ответы и пояснения
- Вызовы инструментов и аргументы функций
- Расширенное мышление (если включено)

У этого подхода есть два основных преимущества:

- Он не требует включения мышления.
- Он может влиять на весь расход токенов, включая вызовы инструментов. Например, более низкий уровень effort означает, что Claude будет делать меньше вызовов инструментов. Это даёт гораздо большую степень контроля над эффективностью.

### Рекомендуемые уровни effort для Sonnet 4.6

Sonnet 4.6 по умолчанию использует уровень effort `high`. Явно задавайте effort при использовании Sonnet 4.6, чтобы избежать неожиданной задержки:

- **Medium effort** (рекомендуемое значение по умолчанию): наилучший баланс скорости, стоимости и производительности для большинства приложений. Подходит для агентного программирования, рабочих процессов с интенсивным использованием инструментов и генерации кода.
- **Low effort:** для высоконагруженных или чувствительных к задержке рабочих нагрузок. Подходит для чата и сценариев, не связанных с программированием, где приоритет отдаётся более быстрому отклику.
- **High effort:** для сложных рассуждений и задач, где качество важнее скорости или стоимости.
- **Max effort:** для задач, требующих абсолютно максимальных возможностей без ограничений на расход токенов.

### Рекомендуемые уровни effort для Claude Opus 4.7

**Начинайте с `xhigh` для программирования и агентных сценариев** и используйте `high` как минимум для большинства рабочих нагрузок, чувствительных к интеллектуальным возможностям. Снижайте до `medium` для рабочих нагрузок, чувствительных к стоимости, или повышайте до `max` только тогда, когда ваши оценки показывают измеримый запас улучшения на уровне `xhigh`.

Значение по умолчанию в API — `high`. Чтобы использовать `xhigh`, задайте `effort` явно; переданное вами значение переопределяет значение по умолчанию.

Claude Opus 4.7 также более строго соблюдает уровни effort, чем Claude Opus 4.6, особенно на уровнях `low` и `medium`. При более низких уровнях effort модель ограничивает свою работу тем, что было запрошено, не выходя за рамки задания. Если вы наблюдаете поверхностные рассуждения по сложным задачам с Claude Opus 4.7, повысьте effort вместо того, чтобы обходить это с помощью подсказок. Если вам необходимо сохранить низкий effort ради задержки, добавьте целевые указания вроде «Эта задача требует многоэтапных рассуждений. Тщательно подумайте перед ответом».

При запуске Claude Opus 4.7 с уровнем effort `xhigh` или `max` задавайте большое значение `max_tokens`, чтобы у модели было пространство для размышлений и действий через субагентов и вызовы инструментов. Разумное значение по умолчанию — начать с 64k токенов и далее настраивать.

### Рекомендуемые уровни effort для Claude Opus 4.8

Рекомендации для Claude Opus 4.7 также применимы к Claude Opus 4.8. **Начинайте с `xhigh` для программирования и агентных сценариев**, используйте `high` для большинства других рабочих нагрузок, чувствительных к интеллектуальным возможностям, и снижайте до `medium` или `low` только тогда, когда вы измерили, что более низкий уровень сохраняет качество на ваших оценках.

Значение по умолчанию — `high` на всех платформах, включая Claude API и Claude Code. Задайте `effort` явно, чтобы использовать другой уровень; переданное вами значение переопределяет значение по умолчанию.

При запуске Claude Opus 4.8 с уровнем effort `xhigh` или `max` задавайте большое значение `max_tokens`, чтобы у модели было пространство для размышлений и действий через субагентов и вызовы инструментов. Разумное значение по умолчанию — начать с 64k токенов и далее настраивать.

### Рекомендуемые уровни effort для Claude Fable 5

Effort — это основной инструмент управления балансом между интеллектуальными возможностями, задержкой и стоимостью в Claude Fable 5. **Начинайте с `high` (значение по умолчанию) для большинства задач**, используйте `xhigh` для рабочих нагрузок, наиболее чувствительных к возможностям модели, и снижайте до `medium` или `low` для рутинной работы. Более низкие настройки effort в Claude Fable 5 по-прежнему дают хорошие результаты и часто превосходят производительность `xhigh` на предыдущих моделях. При `high` и `xhigh` задавайте большое значение `max_tokens`: это жёсткий лимит на общий вывод — мышление плюс текст ответа. См. [Контроль затрат](/docs/ru/build-with-claude/adaptive-thinking#cost-control).

Снижайте effort, если задача выполняется, но занимает больше времени, чем необходимо, или если вы хотите более быстрый, интерактивный стиль работы. Те же рекомендации применимы к Claude Mythos 5. Более подробные рекомендации см. в разделе [Подсказки для Claude Fable 5](/docs/ru/build-with-claude/prompt-engineering/prompting-claude-fable-5).

## Базовое использование

```
`client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-opus-4-8",
    max_tokens=4096,
    messages=[
        {
            "role": "user",
            "content": "Analyze the trade-offs between microservices and monolithic architectures",
        }
    ],
    output_config={"effort": "medium"},
)

print(response.content[0].text)`
```

## Когда настраивать параметр effort

- Используйте **max effort**, когда вам нужны абсолютно максимальные возможности без ограничений: наиболее тщательные рассуждения и глубочайший анализ. Доступен в Claude Fable 5, Claude Mythos 5, Claude Opus 4.8, Claude Mythos Preview, Claude Opus 4.7, Claude Opus 4.6 и Claude Sonnet 4.6.
- Используйте **xhigh effort** для продвинутого программирования и сложной агентной работы, требующей расширенного исследования, например многократных вызовов инструментов и детального поиска. Доступен в Claude Fable 5, Claude Mythos 5, Claude Opus 4.8 и Claude Opus 4.7.
- Используйте **high effort** (значение по умолчанию) для сложных рассуждений, тонкого анализа, трудных задач программирования или любых задач, где качество важнее скорости или стоимости.
- Используйте **medium effort** как сбалансированный вариант, когда вы хотите получить надёжную производительность без полного расхода токенов, характерного для high effort.
- Используйте **low effort**, когда вы оптимизируете скорость (поскольку Claude отвечает меньшим количеством токенов) или стоимость. Например, для простых задач классификации, быстрых справок или высоконагруженных сценариев, где незначительные улучшения качества не оправдывают дополнительную задержку или расходы.

**Режим ultracode в Claude Code:** ultracode отображается в меню effort в Claude Code, но это не дополнительный уровень effort в API. Значения, описанные на этой странице, — это полный набор, который принимает API. Ultracode сочетает уровень effort `xhigh` с постоянным разрешением для Claude Code запускать многоагентные рабочие процессы, предоставляемым через [системные сообщения в середине разговора](/docs/ru/build-with-claude/mid-conversation-system-messages). Чтобы реализовать аналогичное поведение с помощью API, см. [Создание режима оркестрации](/docs/ru/build-with-claude/mid-conversation-effort-example).

## Effort и использование инструментов

При использовании инструментов параметр effort влияет как на пояснения вокруг вызовов инструментов, так и на сами вызовы инструментов. Более низкие уровни effort, как правило:

- Объединяют несколько операций в меньшее число вызовов инструментов
- Делают меньше вызовов инструментов
- Переходят непосредственно к действию без предисловий
- Используют краткие подтверждающие сообщения после завершения

Более высокие уровни effort могут:

- Делать больше вызовов инструментов
- Объяснять план перед выполнением действий
- Предоставлять подробные сводки изменений
- Включать более исчерпывающие комментарии к коду

## Effort и расширенное мышление

Параметр effort работает совместно с расширенным мышлением. Его поведение зависит от модели:

- **Claude Fable 5 и Claude Mythos 5** используют [адаптивное мышление](/docs/ru/build-with-claude/adaptive-thinking), которое всегда включено (конфигурация `thinking` не требуется). `thinking: {type: "disabled"}` отклоняется. Effort управляет глубиной мышления так же, как в Opus 4.8 и Opus 4.7.
- **Claude Opus 4.8** использует [адаптивное мышление](/docs/ru/build-with-claude/adaptive-thinking) (`thinking: {type: "adaptive"}`), где effort является рекомендуемым способом управления глубиной мышления. Ручное расширенное мышление (`thinking: {type: "enabled", budget_tokens: N}`) не поддерживается и возвращает ошибку 400. Модель сама решает, когда и сколько размышлять, исходя из каждого запроса, поэтому она запускает мышление только по мере необходимости. При уровнях effort `high`, `xhigh` и `max` Claude почти всегда глубоко размышляет. При более низких уровнях он может пропускать мышление для более простых задач. Установите `thinking: {type: "adaptive"}`, чтобы включить мышление; без этого запросы выполняются без мышления.
- **Claude Mythos Preview** использует [адаптивное мышление](/docs/ru/build-with-claude/adaptive-thinking) по умолчанию (конфигурация `thinking` не требуется). `thinking: {type: "disabled"}` отклоняется. Effort управляет глубиной мышления так же, как в Opus 4.7 и Opus 4.6.
- **Claude Opus 4.7** использует [адаптивное мышление](/docs/ru/build-with-claude/adaptive-thinking) (`thinking: {type: "adaptive"}`), где effort является рекомендуемым способом управления глубиной мышления. Ручное расширенное мышление (`thinking: {type: "enabled", budget_tokens: N}`) больше не поддерживается в Opus 4.7; вместо этого используйте адаптивное мышление с effort. При уровнях effort `high`, `xhigh` и `max` Claude почти всегда глубоко размышляет. При более низких уровнях он может пропускать мышление для более простых задач.
- **Claude Opus 4.6** использует [адаптивное мышление](/docs/ru/build-with-claude/adaptive-thinking) (`thinking: {type: "adaptive"}`), где effort является рекомендуемым способом управления глубиной мышления. Хотя `budget_tokens` всё ещё принимается в Opus 4.6, он считается устаревшим и будет удалён в одном из будущих выпусков. При уровнях effort `high` и `max` Claude почти всегда глубоко размышляет. При более низких уровнях он может пропускать мышление для более простых задач.
- **Claude Sonnet 4.6** использует [адаптивное мышление](/docs/ru/build-with-claude/adaptive-thinking) (где effort управляет глубиной мышления). Ручное мышление с [чередующимся режимом](/docs/ru/build-with-claude/extended-thinking#interleaved-thinking) (`thinking: {type: "enabled", budget_tokens: N}`) всё ещё работает, но считается устаревшим.
- **Claude Opus 4.5** использует ручное мышление (`thinking: {type: "enabled", budget_tokens: N}`), где effort работает совместно с бюджетом токенов мышления. Установите уровень effort для вашей задачи, затем задайте бюджет токенов мышления в зависимости от сложности задачи.

Параметр effort можно использовать как с включённым расширенным мышлением, так и без него. При использовании без мышления он по-прежнему управляет общим расходом токенов для текстовых ответов и вызовов инструментов.

## Рекомендации

- **Задавайте effort явно:** API по умолчанию использует `high`, но правильная отправная точка зависит от вашей модели и рабочей нагрузки.
- **Используйте low для чувствительных к скорости или простых задач:** когда важна задержка или задачи просты, low effort может значительно сократить время ответа и затраты.
- **Тестируйте свой сценарий использования:** влияние уровней effort варьируется в зависимости от типа задачи. Оцените производительность на ваших конкретных сценариях перед развёртыванием.
- **Рассмотрите динамический effort:** настраивайте effort в зависимости от сложности задачи. Простые запросы могут требовать low effort, тогда как агентное программирование и сложные рассуждения выигрывают от high effort.
